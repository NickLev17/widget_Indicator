# Вашему вниманию представлен виджет индикации ошибки
## Принцип работы виджета
У виджета есть три состояния: неактивное, активное, состояние ошибки. При запуске приложения состояния виджета по умолчанию неактивное.
### Неактивное состояние виджета:
-  цвет индикатора серый
-  при нажатии на виджет левой кнопкой мыши, событие игнорируется
-  при нажатии правой кнопкой мыши, пользователю будет предоставлен ряд всплывающих диалоговых окон, для задания цветовых состояний виджета
### Активное состояние:
-  цвет индикатора зеленый
-  при нажатии на виджет левой кнопкой мыши, пользователю выводится диалоговое окно с информацией об индикаторе (состояние (исправно) и что характеризует данный индикатор)
-  при нажатии правой кнопкой мыши, пользователю будет предоставлен ряд всплывающих диалоговых окон, для задания цветовых состояний виджета
### Состояние ошибки:
-  цвет индикатора красный
-  при нажатии на виджет левой кнопкой мыши, пользователю выводится диалоговое окно с критической информацией об индикаторе (состояние (неисправно) и что нужно предпринять для устранения ошибки)
-  при нажатии правой кнопкой мыши, пользователю будет предоставлен ряд всплывающих диалоговых окон, для задания цветовых состояний виджета
## У данного виджета имеется набор различных свойств:
- STATE состояние виджета (активное, неактивное, состояние ошибки)
- name имя ошибки (при желании, указывается пользователем под лампочкой индикации ошибки)
- color_text цвет текста имени ошибки
- color_On цвет активного состояния виджета
- color_Off цвет неактивного состояния виджета
- color_Error цвет состояния ошибки
- frame позволяет пользователю задать цвет рамки виджета
## Набор методов для задания сообщения об ошибке, или же о ее отсутствии
- setInformMessage пользователь указывает, какое информационное сообщение будет выведено на экран, в случае отсутствия ошибки
- setErrorMessage пользователь указывает, какое критическое сообщение будет выведено на экран, в случае ошибки
## Другие методы 
- dafaultInitialisation инициализация параметров виджета по умолчнию
- updateStatePainter вызывает метод update для перерисовки виджета
- набор геттеров и сеттеров свойств
## Набор сигналов
- changeState сигнал изменения состояния виджета
- changeColorText сигнал изменения цвета имени ошибки
- changeFrame сигнал изменения рамки виджета
- changeColorState сигнал изменения цвета лампочки индикации ошибки
- changeName сигнал изменения имени ошибки
## Переопределение виртуальных методов QWidget
- paintEvent отрисовка виджета
- mouseDoubleClickEvent двойной щелчек мышью в пределах области видимости виджета, выводит диалоговое окно с информацией о состоянии индикатора и рекомендации по устранению ошибки, при ее наличии
- mousePressEvent нажатие кнопки мыши в пределах области видимости виджета. При нажатии правой кнопки мыши, выводится ряд диалоговых окон для задания цветовых настроек состояний виджета
